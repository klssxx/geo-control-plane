# Política de Seguridad

## Alcance

Este repositorio es una plantilla de documentación y validación. No debe contener secretos, credenciales, volcados de ejecución, prompts privados, respaldos privados ni configuración específica de máquina.

## Antes de publicar

Ejecuta una auditoría de secretos antes del primer push público. Como mínimo, inspecciona:

- claves de API
- tokens
- contraseñas
- rutas de archivo privadas que revelen datos sensibles
- volcados de ejecución
- registros
- credenciales

Se recomienda el escaneo de secretos y la protección de push de GitHub para repositorios públicos.

## Reportar problemas

Abre un issue en GitHub para errores de documentación, ejemplos inseguros, orientación de reversión faltante o contenido sensible accidental.
